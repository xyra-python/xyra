Swagger/OpenAPI Documentation

Xyra can automatically generate interactive API documentation for your application using Swagger UI. This is a powerful feature for developing and testing your API.

Enabling Swagger

To enable Swagger, you need to provide swagger_options when creating the App instance. The title and version are required.


from xyra import App

swagger_options = {
    "title": "My Awesome API",
    "version": "1.0.0",
    "url": "/docs"  # The URL to access the Swagger UI
}

app = App(swagger_options=swagger_options)

Documenting Routes

Xyra generates API documentation from the docstrings of your route handler functions. The docstrings should be in YAML format.


from xyra import App, Request, Response

app = App(swagger_options={"title": "My API", "version": "1.0.0"})

@app.get("/users")
def get_users(req: Request, res: Response):
    """
    summary: Get all users
    description: Returns a list of all users in the system.
    responses:
      200:
        description: A list of users.
    """
    res.json([{"id": 1, "name": "John Doe"}])

@app.post("/users")
def create_user(req: Request, res: Response):
    """
    summary: Create a new user
    requestBody:
      required: true
      content:
        application/json:
          schema:
            type: object
            properties:
              name:
                type: string
    responses:
      201:
        description: User created successfully.
    """
    # Logic to create a user
    res.status(201).json({"message": "User created"})

Accessing the Documentation

Once your application is running, you can access the Swagger UI at the URL you specified in swagger_options (e.g., http://localhost:8000/docs).

The interactive documentation allows you to see your API endpoints, parameters, and response models, and even send test requests directly from your browser.
