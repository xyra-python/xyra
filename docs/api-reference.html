API Reference

This section provides a detailed reference for the Xyra framework's API.

App Class

The App class is the main entry point of a Xyra application.


class App:
    def __init__(self, options=None, templates_directory: str = "templates", swagger_options: dict = None):
        ...
  • options: Optional configuration for the underlying socketify app.
  • templates_directory: The directory where your HTML templates are stored.
  • swagger_options: A dictionary of options to configure Swagger/OpenAPI documentation.

App Methods

  • listen(port: int, host: str = "localhost", reload: bool = False, logger: bool = False): Starts the web server.
  • get(path: str, handler: callable = None): Decorator to register a GET route.
  • post(path: str, handler: callable = None): Decorator to register a POST route.
  • put(path: str, handler: callable = None): Decorator to register a PUT route.
  • delete(path: str, handler: callable = None): Decorator to register a DELETE route.
  • patch(path: str, handler: callable = None): Decorator to register a PATCH route.
  • options(path: str, handler: callable = None): Decorator to register an OPTIONS route.
  • head(path: str, handler: callable = None): Decorator to register a HEAD route.
  • websocket(path: str, handlers: dict = None): Decorator to register a WebSocket route. Can also be used as a method with a dictionary of handlers.
  • use(middleware: callable): Adds a middleware to the application.
  • static_files(path: str, directory: str): Serves static files from a directory.

Request Class

The Request class represents an incoming HTTP request.


class Request:
    ...

Request Properties

  • params: A dictionary of route parameters.
  • query: A dictionary of query parameters.
  • headers: A dictionary of request headers.
  • method: The HTTP method of the request.
  • url: The URL of the request.
  • body: The raw request body as bytes.
  • app: The application instance.

Request Methods

  • json(): Asynchronously reads the request body as JSON.
  • form(): Asynchronously reads form data from the request body.

Response Class

The Response class is used to create an HTTP response.


class Response:
    ...

Response Methods

  • status(code: int): Sets the HTTP status code.
  • header(name: str, value: str): Sets a response header.
  • text(content: str): Sends a plain text response.
  • json(data: dict): Sends a JSON response.
  • html(content: str): Sends an HTML response.
  • render(template_name: str, **context): Renders an HTML template.
  • redirect(url: str, status_code: int = 302): Redirects the client to a different URL.
